# Multi-View Translator

This skill helps Product Owners (POs) bridge the communication gap between technical teams, product stakeholders, and business leadership. It takes a single topic and "translates" it into three distinct perspectives, ensuring that the message resonates with the specific values of each group.

## Core Perspectives

### 1. Technical View
**Focus:** Feasibility, scalability, and long-term health.
**Stakeholders:** Developers, Architects, DevOps, Security.
**Values:** Code quality, performance, security, maintainability, architectural integrity.

### 2. Product View
**Focus:** User value, functionality, and market fit.
**Stakeholders:** UX Designers, Product Managers, Customer Support, Marketing.
**Values:** User experience, feature completeness, roadmap alignment, competitive advantage.

### 3. Business View
**Focus:** ROI, strategic alignment, and risk management.
**Stakeholders:** Executives, Sales, Finance, Legal, Partners.
**Values:** Revenue growth, cost reduction, market share, legal compliance, strategic milestones.

## Workflow

1.  **Ingest Topic:** Gather details about the feature, issue, or decision.
2.  **Identify Stakeholder Values:** Refer to [stakeholder-profiles.md](references/stakeholder-profiles.md) to understand what each group prioritises.
3.  **Translate Content:** Rewrite the topic for each of the three views.
4.  **Highlight Alignment:** Explicitly state how the views align to support the overall goal.
5.  **Apply Template:** Format the output using the [communication-template.md](assets/communication-template.md).

## Guidelines

- **Avoid Jargon:** While Technical View can use some technical terms, keep Business and Product views focused on outcomes.
- **Emphasise "Why":** For each group, explain why this topic matters *to them*.
- **Maintain Consistency:** The core message must remain the same across all views; only the emphasis and language change.
