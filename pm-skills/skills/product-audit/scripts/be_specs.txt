none1. System overviewFX Management is a service used to store and retrieve foreign exchange (FX) rates used in payment transactions involving currency conversions. It is designed to support integration with other payment providers that don't offer dynamic FX quotations.Note: This service can expose external public API for other systems to integrate and get currency exchange rates from the Zalopay system.1.1 PurposeProvide centralized FX rate management for payment processingEnable reliable currency conversion capabilitiesSupport third-party integrations with standardized APIsEnsure cost-effective and highly available FX rate services1.2 ScopeReal-time and historical rate queriesExternal provider integrationAudit and compliance logging2. ObjectivesEnable storing (inserting) FX ratesProvide reliable APIs to retrieve FX rates based on appId and currency pair.Minimize external provider costs through intelligent cachingEnsure high availability despite external provider limitationsMaintain audit trails for compliance and monitoring3. Requirement and Implementation3.1 System requirement specification (SRS)Functional Requirements:The system must provide the latest exchange rate for the requested currency pair.The system shall allow authorized users to insert exchange rate.The system don't allow user update exchange rate over 10% compare to old state.The system shall automatically fetch provider quotes every 10 minutes. (future feature)The system shall notify admins if provider data is not updated for 1 hour. (future feature)Non-Functional Requirements:PerformanceThe exchange rate query API shall respond within 300 milliseconds under normal loadThe system shall handle up to 30 concurrent requests per secondSupport up to 2 million requests per dayCache hit ratio should be > 95%AvailabilityThe system shall be available 24/7 with 99.99% SLAImplement graceful degradation when external providers are unavailable ( can use the latest FX Rate of the last day based the agreement of OP and business). Almost zero-downtime deployment capabilityScalabilityHorizontal scaling supportDatabase read replicas for query optimizationLoad balancing across multiple instancesCompliance & AuditingThe system must log all insert operations for auditingMaintain compliance with financial regulationsData retention policies for historical records3.2 Problem AnalysisExternal Provider Challenges:The external FX service costs $0.01 per callThe service has only 99% availability rateNeed to minimize costs while maintaining service qualitySolutions:Implement aggressive caching strategiesUse circuit breaker patterns for resilience (potential)Implement retry mechanisms with exponential backoff ( potential) Create fallback mechanisms for high availability3.3.
